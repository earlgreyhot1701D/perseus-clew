# Roadmap

This roadmap is a living document. Dates are intentions, not promises.

---

## Now (shipped)

- **The frontend URL scan (free tier):** six-category agent-readiness scan with AI hero-line and agent simulation. Paste a URL, see what an agent experiences.
- **The 50-site benchmark study:** published dataset, pre-registered predictions, and methodology. 50 sites across 5 verticals, scores and findings public for anyone to audit.

## Next

- **API spec scanning in the UI:** the engine already scores OpenAPI specs (backend-only today). Bringing it to the scan UI so teams can test their API alongside their frontend. The "Team tier" feature.
- **Plain-language findings:** translate technical findings for non-engineer builders. Lead with what an agent can't do, subordinate the technical detail. Same "findings, not fixes" rule.
- **Finding location:** show where in the served HTML each finding occurs. Honest framing: served-page location, not source-code mapping.

## Later / exploring

- **An active agent that drives a browser:** clicks, navigates, fills forms, tests the interactive surface. Not just reasoning over static HTML. Needs a JS-rendering engine and a different trust model.
- **Authenticated / "inside the login" scanning:** the v3 paid tier. Gated on earned trust and the JS-rendering engine. Agents meet your product at public doors first, but the product behind login is the real surface.
